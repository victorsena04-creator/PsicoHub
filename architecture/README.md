# Arquitetura - Projeto PsicoHub

## Camada 1: Arquitetura (POPs)

Esta pasta contém os Procedimentos Operacionais Padrão (POPs) técnicos.

### Estrutura Recomendada

- `README.md` - Este arquivo
- `*.md` - POPs individuais para cada funcionalidade

### Regras

1. POPs devem ser escritos em Markdown
2. Devem definir: objetivos, entradas, lógica de ferramentas e casos de borda
3. **Regra de Ouro**: Se a lógica mudar, atualize o POP antes de atualizar o código
